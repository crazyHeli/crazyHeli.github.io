Game rules

This is funny and amazing game, where your main task is to get as many points as possible!

At the same time you have to avoid collisions with clouds, unidentified flying objects and angry monsters, which spitting fire. It is not so easy as you can think!With each level enemies will move faster and faster!At the beginning of the game you will have 3 lives, each of which you can lose because of 1 clash with enemy objects, mountains or ground.

On your way to the goal you will meet coins and other objects, that will give you points or lives or other super capabilities. With your enemies your can fight using your own bullets. For each target hit you'll get points!At some moment you can find yourself in a wonderful situation without enemies, where you'll be able just fly and collect coins!

Issues

First

At the second transition to the game in game the sequence of coins continued to be drawn from the moment of the exit from the past game in game

Solution

After exit from the game in game we must clear all sequences of coins

Second

Initially, clouds and coins were realized as follows: the cloud is initialized, then after the missile hits it, the cloud goes into a temporary array, then when this array is sorted, coins are created that are transferred to a new array of coins. With this implementation, after hitting the cloud, the remaining clouds in the canvas blinked.

Solution

The logic of coin initialization was rewritten. When you hit the cloud, in the class Collisions, coins are created and placed in an array (without using a temporary array). After that, the clouds stopped blinking.

Third

If you specify one animation cycle, then the object is not cleaned until the animation has passed completely, and yet another collision is called.

Solution

An additional property was added to the object so that other objects could not collide with it until it was fully cleaned
